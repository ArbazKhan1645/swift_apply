# Contributing to SwiftApply

First off, thank you for considering contributing to SwiftApply! It's people like you that make this tool better for everyone.

## How Can I Contribute?

### Reporting Bugs
- Check the [Issues](https://github.com/ArbazKhan1645/swift_apply/issues) page to see if the bug has already been reported.
- If not, open a new issue. Use a clear title and provide as much detail as possible, including steps to reproduce the bug.

### Suggesting Enhancements
- Check the Issues page for existing suggestions.
- Open a new issue with the "enhancement" label. Describe the feature you'd like to see and why it would be useful.

### Pull Requests
1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. If you've changed APIs, update the documentation.
4. Ensure the test suite passes.
5. Make sure your code follows the existing style (run `flutter format .`).
6. Issue that Pull Request!

## Style Guide
- Follow the official [Dart Style Guide](https://dart.dev/guides/language/effective-dart/style).
- Use descriptive variable and function names.
- Keep components modular and reusable.
